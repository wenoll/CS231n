# Dropout 实验总结（CS231n Assignment 2）
## 一、实验背景与目的
本实验是 CS231n 课程作业的一部分，旨在通过从零实现 Dropout 层，验证其原理，并通过对比实验直观理解 Dropout 作为正则化技术在神经网络中的作用。实验分为三个核心环节：实现 Dropout 的前向/反向传播、为全连接网络添加 Dropout 层、对比有无 Dropout 的模型表现。

## 二、核心知识点回顾
### 1. Dropout 的核心原理
Dropout 是一种**防止过拟合的正则化技术**，其核心思想是在训练过程中，随机将一部分神经元的输出置为0，避免神经元之间的共适应（co-adaptation），从而迫使网络学习更通用的特征。

本实验采用**倒置 Dropout（Inverted Dropout）**，关键在于：
- **训练阶段**：以概率 `p` 保留神经元，同时将保留的输出除以 `p`，保证训练和测试阶段的输出期望一致。
- **测试阶段**：不丢弃任何神经元，直接返回输入，无需额外缩放。

### 2. Dropout 的前向传播
```python
def dropout_forward(x, dropout_param):
    p, mode = dropout_param["p"], dropout_param["mode"]
    if mode == "train":
        mask = np.random.rand(*x.shape) < p  # 生成0/1掩码
        out = x * mask / p  # 除以p缩放，保证期望不变
    else:  # test模式
        out = x
    cache = (dropout_param, mask)
    return out.astype(x.dtype, copy=False), cache
```
- `mask`：与输入形状相同的开关矩阵，True=保留、False=丢弃，是 Dropout 的核心控制单元。
- `/p` 的作用：训练时缩放输出，避免测试时数值期望发生变化，保证训练和测试分布一致。

### 3. Dropout 的反向传播
```python
def dropout_backward(dout, cache):
    dropout_param, mask = cache
    mode = dropout_param["mode"]
    if mode == "train":
        dx = dout * mask / dropout_param["p"]  # 仅对保留的神经元传递梯度
    else:
        dx = dout
    return dx
```
反向传播仅对训练时未被丢弃的神经元传递梯度，同时保持与前向传播一致的缩放系数 `/p`，保证梯度计算的正确性。

### 4. Dropout 在全连接网络中的应用
在 `FullyConnectedNet` 中，Dropout 被添加在**每一层 ReLU 激活函数之后**（题目要求），网络结构变为：
`affine → (batchnorm/可选) → relu → dropout → affine → ...`
- 当 `dropout_keep_ratio != 1` 时启用 Dropout，否则不使用。
- 训练/测试模式自动切换，Dropout 在测试阶段自动关闭。

## 三、实验步骤与结果分析
### 步骤1：Dropout 层实现与验证
通过测试代码验证 Dropout 层的正确性，结果显示：
- 训练阶段输出均值与输入均值几乎一致，证明 `/p` 缩放有效。
- 被置0的神经元比例约为 `1-p`，符合 Dropout 的随机丢弃逻辑。
- 反向传播梯度检查误差为 `1.89e-11`，远小于 `1e-8` 的合格标准，证明梯度计算完全正确。

### 步骤2：带 Dropout 的全连接网络梯度检查
对不同 Dropout 保留概率（1.0、0.75、0.5）的模型进行梯度检查，结果显示：
- 所有参数的梯度相对误差均小于 `1e-6`，证明 Dropout 与全连接网络的集成无错误。
- 不同 Dropout 强度下，梯度计算稳定，模型可正常训练。

### 步骤3：过拟合对比实验
使用 500 个 CIFAR-10 训练样本（数据量小，易过拟合），对比无 Dropout（p=1.0）和有 Dropout（p=0.25）的模型表现：

| 指标                | 无 Dropout（蓝色）       | 有 Dropout（橙色）       |
|---------------------|--------------------------|--------------------------|
| 训练准确率          | 接近100%，几乎完全拟合   | 约94%，未完全拟合        |
| 验证准确率          | 后期约0.28-0.30，波动下降 | 最高约0.34，整体更高更稳定 |
| 过拟合程度          | 明显过拟合（训练/验证差距大） | 过拟合显著抑制（差距缩小） |

实验结果直观证明：Dropout 有效降低了训练准确率，提升了验证准确率，缩小了训练与验证表现的差距，是抑制过拟合、提升泛化能力的有效手段。

## 四、关键问题解答
### 1. 不除以 `p` 会发生什么？
如果训练阶段不除以 `p`，输出数值会按比例 `p` 缩小，导致训练和测试阶段的数值期望不一致，网络无法学习稳定的特征，训练过程会崩溃，预测效果极差。

### 2. Dropout 作为正则化的作用是什么？
Dropout 是一种有效的正则化手段，通过随机丢弃神经元，迫使网络学习更通用的特征，避免对训练数据的过拟合，从而显著提升模型在未见过的验证数据上的泛化能力。

---

## 五、实验结论
1. **Dropout 原理正确**：倒置 Dropout 通过训练时的随机丢弃和 `/p` 缩放，保证了训练与测试阶段的输出分布一致，实现了正确的前向和反向传播。
2. **Dropout 能有效抑制过拟合**：对比实验显示，使用 Dropout 的模型训练准确率略低，但验证准确率更高、更稳定，泛化能力显著提升。
3. **Dropout 易于集成**：在全连接网络中，Dropout 可直接添加在 ReLU 激活之后，与网络的前向/反向流程无缝结合，且不影响梯度计算的正确性。

---

## 六、知识点
- Dropout 的本质：通过随机化训练过程，模拟多个不同网络的集成效果，从而降低模型方差。
- 应用场景：Dropout 常用于全连接层和卷积层（但在卷积层中通常使用 Dropout 的变体如 DropConnect），尤其适用于数据量较小、模型容量较大的场景。
- 注意事项：Dropout 会增加训练时间，且在测试阶段必须关闭，否则会导致模型表现异常。
